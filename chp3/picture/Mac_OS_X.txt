
 3-5 Mac OS Xへのインストール

　Firebirdでは、InterBaseがサポートしていなかったMac OS X版があり、Mac OS X 10.1以降とその基盤となっているDarwinをサポートしています。Firebirdの公開当初にリリースされていたものは、ターミナルからコマンドラインでインストールを行う形になっていましたが、その後にリリースされたMac OS X 10.2に対応したGUIインストーラ版のFirebird 1．0.1がリリースされています。このバージョンでは、Mac OS X 10.2で起きていたインストール時に止ってしまう不具合も解消されています。また、Mac OS X 10.1以前や、ネイティブのDarwinで使用する場合には、従来のFirebird 1.0を使うことが奨められています。
　ここれでは、最新のFirebird 1.0.1を使ったインストールを紹介しましょう。

3-5-1

　CD-ROM内のfirebird-1.0.1CS.pkgをダブルクリックして、インストーラを起動します。インストールには管理者権限が必要になるので、認証のダイアログが表示され、パスワードの入力が必要になります。パスワードを入力すると、紹介画面になります（図3-5-1）。

（図3-5-1　Firebird 1.0.1のインストーラ）

　「続ける」を押すと、「大切な情報」として、以下のようなインストールノートが表示されます。

（図3-5-2　「大切な情報」の画面）

インストールノート（日本語訳）
-----------------------------------------------------------------------------

Firebirdのインストールにおいて、「firebird」という新しいユーザーを作成します。これは、セキュリティを追加するためです。明示的に行おうとする場合以外には、このユーザーを削除してはいけません。インストーラは、/Library/Frameworks以下にFirebirdのフレームワークをインストールします。デフォルトでは、スーパーユーザーとして「SYSDA」を「masterkey」というパスワードでインストールします。ドキュメントを参考にして、gsecを使って、必ずパスワードを変更してください。

すべての標準的なコマンドラインツールは、/Library/Frameworks/Firebird.framework/Resources/bin以下にインストールされます。現在のところ、Firebird用のGUIコンソールアプリケーションは提供されていません。これについては作業中です。さらに、Objective-Cのインターフェイスについても作業中です。これらの取り組みについて、援助することに興味があるならば、私たちにコンタクトしてください（コンタクト情報は、Firebirdnoサイトに提示されています）。

データベースにアクセスさせようとするすべてのMac OS Xユーザーは、.gdbファイルで、＊必ず＊、書き換え可能なアクセス権を与えなければならないことに注意してください。

完全なリリースノートは、FirebirdのWebサイト（www.firebirdsql.org）で見ることができるでしょう。Firebirdに関するドキュメントは、www.interbase.comで提供されているInterBaseのドキュメントとほとんど同じです。Firebirdに関して、何か問題があった場合には、「ib-support」というYahoo! Groupがあります。また、IBPhoenix社のWebサイト（www.ibphoenix.com）も大いに参考になるでしょう。

-----------------------------------------------------------------------------

　「続ける」を押すと、使用許諾の画面になります。これは、Windows版で表示される「InterBase Public Liscense」と同じものです。ここで、さらに「続ける」を押すと、「使用許諾契約」の同意を求めるダイアログが表示されるので、「同意する」を押して続けます。

（図3-5-3　「使用許諾契約」の画面）

　次に、「インストール先の選択」画面が表示されます。ここで選択できるのは、どのドライブにインストールするかだけで、Firebirdは、選択したドライブの/Library/Frameworks/Firebird.framework/Resources/bin以下にインストールされます。また、インストールするには、約10MBのディスク容量が必要になります。

（図3-5-4　「インストール先を選択」の画面）

　「続ける」を押すと、「インストールの種類」の画面になります。特にオプションなどの設定はないので、「簡易インストール」が自動的に選択されています。「カスタマイズ」はディスイネーブルになっており、この画面で「インストール」を押せば、インストールは自動的に進んで完了します。

（図3-5-5　「インストールの種類」の画面）
（図3-5-6　「インストール中」の画面）

最後にシステムの最適化が行われて、インストールは完了します。これで、Firebirdのインストールを無事終えることができました。「NetInfoマネージャ」（[アプリケーション]→[Utilities]→[NetInfo Maneger]）で、[groups]と[users]のそれぞれに、「firebird」が追加されていることが確認できます。

（図3-5-7　「仕上げ」の画面）
（図3-5-8　「NetInfoマネージャ」の画面）　※2つのダイアログを組み合わせてください

　しかし、今のところ、Mac OS X版では、IBConsoleのようなGUI管理ツールは提供されていません。これについては、現在、作業中であり、Windows版のインストールAPIと同じようなObjettive-Cによるインターフェイスについても作業が進められているようです。また、/Library/Frameworks/Firebird.framework/Resources/bin以下に主なコマンドラインツールが収録されているので、現状では、これらを使って管理を行う必要があります。
